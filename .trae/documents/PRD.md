## 1. 产品概述

个人健身计划管理应用，帮助用户按肌肉群组织训练计划，细化到每个部位的具体动作与组数，并通过日历视图追踪训练完成情况与完成度。
- 目标用户：有健身习惯、希望系统化管理训练计划的个人用户
- 价值：将"计划制定—日历查看—完成度追踪"闭环打通，让训练有据可依、进度可视化

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|---------------------|------------------|
| 个人用户 | 本地免登录（数据存浏览器） | 创建/编辑/删除计划，标记训练完成度，查看日历 |

### 2.2 功能模块
1. **日历总览页**：月历视图，按日显示训练状态与完成度，点击日期查看当日计划详情
2. **计划管理页**：创建/编辑/删除训练计划，按肌肉群组织，细化到动作与组数
3. **训练详情页**：查看某日计划的所有动作，逐项标记完成情况与完成度

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 日历总览页 | 顶部统计栏 | 显示本月训练天数、计划总组数、完成组数、平均完成度 |
| 日历总览页 | 月历视图 | 按日显示训练标记（计划数、完成度色块），点击切换日期 |
| 日历总览页 | 当日计划卡片 | 展示选中日期的训练计划列表，含肌肉群、动作数、完成度进度条 |
| 计划管理页 | 计划列表 | 展示所有训练计划卡片，支持编辑/删除/复制 |
| 计划管理页 | 计划编辑抽屉 | 选择肌肉群、添加动作、设置组数/次数/休息时长，选择计划执行日期 |
| 训练详情页 | 动作清单 | 按肌肉群分组展示动作，每个动作可标记完成/部分完成/跳过 |
| 训练详情页 | 完成度统计 | 实时计算并展示当日整体完成度，支持逐组打分 |

## 3. 核心流程

**主流程**：用户在"计划管理页"创建训练计划（选肌肉群→添加动作→设置组数→选日期）→ 计划自动出现在"日历总览页"对应日期 → 训练日点击进入"训练详情页"→ 逐动作标记完成度 → 日历视图实时更新完成度色块。

```mermaid
    "F --> G[日历总览页显示计划标记]"
    "G --> H[训练日点击进入训练详情]"
    "H --> I[逐动作标记完成度]"
    "I --> J[日历视图更新完成度色块]"
```

## 4. 界面设计

### 4.1 设计风格
- **主题**：运动竞技暗色风，高对比、能量感强
- **主色**：近黑底色 #0A0A0B + 电光绿强调色 #C6FF3D + 暖橙警示色 #FF6B35
- **次色**：深灰卡片 #16161A，描边 #2A2A30，文字 #FAFAFA / #9A9AA0
- **按钮**：圆角 8px，主按钮电光绿填充黑字，次按钮描边
- **字体**：标题用 "Bebas Neue" / "Oswald"（运动感强），正文用 "Inter" 备选 "Manrope"
- **布局**：桌面优先，左侧导航 + 右侧主内容；卡片化呈现，留白克制
- **图标**：线性图标风格，肌肉群用色块 + 缩写标识
- **动效**：日历日期切换淡入，完成度进度条平滑过渡，按钮 hover 微位移

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|-----------|-------------|-------------|
| 日历总览页 | 顶部统计栏 | 4 个数据卡片横排，电光绿数字 + 灰色标签 |
| 日历总览页 | 月历视图 | 7 列网格，每个日期格含日期数字 + 完成度色块/圆点 |
| 日历总览页 | 当日计划卡片 | 卡片列表，左侧色条标识肌肉群，右侧进度环 |
| 计划管理页 | 计划列表 | 网格卡片，封面色 + 肌肉群标签 + 动作数 + 日期 |
| 计划管理页 | 编辑抽屉 | 右侧滑出抽屉，分步表单，肌肉群色块选择器 |
| 训练详情页 | 动作清单 | 按肌肉群分组的可展开列表，每行动作含组数指示器 |
| 训练详情页 | 完成度统计 | 顶部大号完成度环 + 完成组数计数 |

### 4.3 响应式
- 桌面优先设计（≥1024px 完整三栏体验）
- 平板（768-1024px）保持双栏，侧边导航收起为图标
- 移动端（<768px）单栏，底部 Tab 切换页面，日历改为竖向列表
- 触控优化：按钮最小 44px 触摸区，滑动切换月份
